# 博客文章配图视觉规范

请为博客文章配图，版式参考 Excalidraw 演示图，但成品输出为干净的 PNG。

## 强约束

### 1. 整体风格

- 整体是浅色背景、低噪音、强留白
- 不要深色科技风
- 不要炫光、渐变、网格、拟物阴影

### 2. 版式结构

- 版式像 Excalidraw 演示图
- 大标题居中或左上，简洁直接
- 核心内容用 3-6 个圆角卡片或模块表达
- 卡片之间留足空白，不能拥挤
- 底部可以有一句总结条

### 3. 视觉语言统一

- 白底或极浅米底
- 卡片使用浅蓝、浅绿、浅粉、浅黄、浅紫、浅灰
- 描边颜色比填充更深，线条清晰
- 不要使用复杂装饰元素

### 4. 文字安全

- 不允许出现任何溢出、贴边、压线、遮挡
- 长标题必须主动换行
- 长正文必须拆成 2-4 行短句
- 每个卡片内边距要大
- 英文长词如 `Context Engineering`、`Harness Engineering` 必须拆行处理

### 5. 箭头和连线

- 只在真正需要表达流程时使用
- 箭头细、小、方向明确
- 不允许出现夸张大箭头
- 不允许让箭头贴着文字

### 6. 信息密度

- 信息结构优先，不追求花哨
- 一张图只表达一个核心观点
- 不要把太多句子塞进一张图
- 每个卡片只保留“标题 + 1 个短解释 + 2-3 个短点”

### 7. 字体气质

- 不使用 Excalidraw 默认手写字
- 改用更干净的现代无衬线风格
- 中文标题接近产品设计图中的黑体或无衬线风格
- 英文模块标题可以带一点等宽字体的工程感
- 标题字重略高，正文常规字重
- 字面要干净、均匀、理性、克制
- 不要俏皮手写感
- 不要粗糙涂鸦字

### 8. 字号层级

- 大标题必须明显大于卡片标题
- 卡片标题必须明显大于正文
- 正文大小必须保证在正文内嵌展示时依然清晰可读
- 不要为了塞入更多内容而把正文字号压得过小
- 同一张图内字号层级应稳定统一，不要忽大忽小
- 同一篇文章的多张配图之间，标题、卡片标题、正文的相对大小应尽量一致
- 如果内容过多，优先删减文案、拆句、增大留白，而不是缩小字号

### 9. 版式与字体组合目标

- 版式上保留 Excalidraw 式的信息卡片结构
- 字体气质更像“产品图解 / 设计说明图”
- 重点是结构清晰，不是手绘感本身
- 可以有轻微手绘描边，但文字必须像正式排版
- 英文长词依旧优先拆行，保证卡片内安全留白

### 10. icon 规则

- 默认不使用 icon
- 只有当 icon 能明显提升理解效率时才使用
- icon 必须服务于解释，不得作为装饰性元素滥用
- 若标题、结构和分组已经足够清晰，则优先不用 icon

### 11. 尺寸与画幅

- 默认优先使用 `16:9` 横版
- `16:9` 更适合概念图、对比图、总结图、3-6 卡片的知识卡片
- 如果内容是长流程、多层级结构、节点较多、横向或纵向链路较长，则不强制 `16:9`
- 不要为了维持 `16:9` 而压缩文字可读性、破坏留白或强行塞满版面

### 12. 输出范围

- 默认只需要最终 PNG
- 不需要 `.excalidraw` 图源，除非用户明确要求

### 13. 内容过多时的处理

- 优先做减法
- 优先拆句、拆行、缩短表达
- 不要强行塞进原布局

### 14. 最终目标

- 看起来像“高质量 Excalidraw 风格知识卡片”
- 更准确地说，是“Excalidraw 式版式 + 产品图解式字体”
- 干净、克制、可读
- 一眼能看懂
- 适合博客正文内嵌展示

## 风格参考边界

- 默认优先使用本文件作为第一风格来源，并结合当前文章内容决定画面表达
- 不得因为仓库中已有其他文章的配图，就把它们作为默认视觉参考
- 如果用户没有明确要求跨文章对齐风格，则应避免把其他文章的构图气质、模块习惯或配色倾向带入当前文章
